# MCP Security Posture Component

MCP is young, adoption is accelerating, and the security model (server trust,
tool permissions, transport security, credential delegation) is still being
figured out in public.

The threats component already references OWASP MCP Top 10. A dedicated
component would evaluate MCP-specific risks — which server patterns are
defensible, what the trust boundaries actually look like, where the spec
has gaps.

The taxonomy disagreements pattern from threats fits perfectly here since
MCP security guidance is fragmented across Anthropic docs, OWASP, and
community posts. First-mover opportunity — nobody has consolidated this
into structured, evaluable knowledge.
